Comunicação do cliente com o servidor


Resumidamente, existe numa ligação um cliente que faz pedidos a um servidor que, por sua vez, lhe responde. Neste cenário é possível considerar o browser como sendo o cliente, ao passo que um Serviço Web remoto pode ser considerado o servidor.

Existem, de resto, protocolos específicos para a comunicação web, sendo um dos mais adotados o HTTP. Existem diversos tipos de pedidos HTTP, entre os quais:

  • GET: Obtém informação;
  • PUT: Usado para atualizar informação;
  • POST: Submete a informação;
  • DELETE: Remove a informação.

De que se trata o URI?

Abreviação para Uniform Resource Identifier, o objetivo do URI é identificar o recurso remoto ao qual se está a tentar aceder.

Exemplo de um identificador URI: http://musicbrainz.org/ws/1/artist/?query=artist:Imagine%20Dragons%fmt=json

Sintaxe geral de um identificador URI: scheme:[//[user:password@]host[:port]][/]path[?query][#fragment]

  • Por uma questão de normalização, o identificador tem que ser codificado de forma a que o servidor o consiga perceber. Como por exemplo, a substituição dos espaços por outra representação: %20;
  • A função encondeURI() facilita o processo acima descrito.

Associação do jQuery com a técnica AJAX


Para além das "infinitas" vantagens em utilizar a extensão de JavaScript denominada de jQuery, também existe uma ligação entre a técnica AJAX (que descreverei em seguida) e a popular biblioteca.

Primeiramente, o que é AJAX?

AJAX trata-se de uma abreviatura para "Asynchronous JavaScript And XML" e essencialmente trata-se de uma técnica utilizada para conseguir carregar informação no browser sem ser necessário recarregar a página em que o utilizador se encontra.

Os métodos de AJAX do jQuery permitem a troca de dados através de pedidos HTTP, entre os quais, GET e POST


Sintaxe para a utilização de AJAX através da biblioteca jQuery


    Pedido AJAX através de jQuery Explicação do pedido
    $.get(URL, callback) Trata-se de um pedido para obter informação através de jQuery
    $.post(URL, data, callback) Trata-se de um pedido para submeter informação através de jQuery

Considerações finais


  • AJAX é uma técnica que me espantou da forma mais positiva possível, pois acaba por ter um impacto enorme na minha visualização dos pedidos do cliente ao servidor;

  • Vejo a principal vantagem da técnica (aplicada, de resto, com jQuery) no facto de me vir a permitir recarregar uma página no browser sem ser necessário dar um "refresh" no mesmo. O que, pelo menos no meu caso em particular, irá ser muito útil em projetos futuros nos quais estarei decerto envolvido;

  • Falando dos próprios pedidos e não da principal vantagem que vejo no método/técnica é, sem dúvida, um enorme benefício poder utilizar AJAX para proceder à implementação tanto de caixas de pesquisa como dos filtros das mesmas.
