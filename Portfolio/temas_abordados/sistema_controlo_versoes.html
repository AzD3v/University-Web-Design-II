Sistema de Controlo de Versões

O que é um Sistema de Controlo de Versões?


Abreviado para SCV, um Sistema de Controlo de Versões tem como objetivo primordial manter um histórico de alterações e permitir a navegação e operações sobre o conteúdo desse referido histórico. Como exemplos de de sistemas deste género são de referir:

  • GIT
  • Subversion
  • Mercurial
  • CVS
  • Bazaar

Os tipos de arquitetura de um SCV são:

  • Centralizada: A sincronização é efetuada através de um único ponto;



  • Distribuída: A sincronização é efetuada através de múltiplos pontos independentes uns dos outros.

A possibilidade de criar "Branches" no GIT


O GIT permite a possibilidade de criar "branches" aquando do versionamento de projetos. De que se trata esta funcionalidade?

Essencialmente, "branching" é a criação de "ramos" do projeto que permitirão alterações que não irão interferir com o "tronco" do projeto. "Ramos" nos quais é facilitado o experimento de novas alterações sem comprometer a integridade total do trabalho em questão (sendo este preservado na condição de master).

Os comandos utilizados para o "management" dos branches são: git branch "nome do branch" (permite criar um novo branch no projeto), git checkout "branch" (mudança entre branches ou para o master), git branch (permite listar todos os branches) e git merge (o acto de juntar um branch a outro ou ao master), todos demonstrados em seguida:
