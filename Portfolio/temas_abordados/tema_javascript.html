Linguagem de Programação JavaScript

Introdução ao JavaScript


O que é a ECMAScript?

  • ECMAScript trata-se de uma norma que especifica as propriedades de uma linguagem de scripting, isto é, uma linguagem interpretada;
  • O objetivo desta norma trata-se de fornecer detalhes e orientações para o desenvolvimento de uma linguagem interpretada compatível com o norma já referida;
  • Este regulamento possui várias versões (ES5, ES6, ES7, entre outras) e o suporte de cada uma das versões varia consoante as implementações distintas dos próprios browsers;
  • O JavaScript é uma linguagem interpretada baseada na especificação ECMAScript.


Interpretador de JavaScript (JavaScript engine):

  • O JavaScript engine trata-se de um programa que compreende e executa código proveniente da linguagem JavaScript;
  • Estando tipicamente embutido em web browsers, também é possível executar JavaScript do lado do servidor, mesmo tendo em conta o facto do lado do cliente ser a principal área de ação da linguagem. No caso do google chrome, é utilizado o interpretador V8, no Mozilla Firefox é utilizado o SpiderMonkey, o Chakra no Microsoft Edge e o JavaScriptCore no Safari/Webkit.

JavaScript Runtime:

  • É essencialmente o ambiente em que o código JavaScript é "traduzido" pelo interpretador de JavaScript em tempo de execução. Como por exemplo, este fornece o acesso ao objeto que permite a manipulação de HTML no browser, sendo que também providencia o ingresso ao objeto que permite a manipulação da própria janela do browser.

A linguagem JavaScript é uma linguagem "multi-paradigma", o que significa que permite tanto Programação Funcional como Programação Orientada a Objetos (OOP).


Sintaxe básica da linguagem JavaScript


Sendo de referir desde já que apresentarei obviamente diversos e variados exemplos da aplicação da sintaxe e semântica desta linguagem de programação aquando da minha análise às fichas de trabalho efetuadas, irei aqui tentar descrever de forma sucinta os aspetos essenciais da escrita de código em JavaScript.

Os programas em JavaScript são executados com declarações, como por exemplo:

Declaração Significado da declaração
var A, B, C; As variáveis A, B e C são declaradas
A = 10; O valor de 10 é atribuído à variável A
B = 5; O valor de 5 é atribuído à variável B
C = A + B; A soma das variáveis A e B é atribuída à variável C

Notas de destaque:

  • O carácter ";" termina uma declaração;
  • A palavra "var" é reservada para a declaração de variáveis, assim como "let" e "const". A palavra "let" permite a declaração de variáveis ao nível do bloco em que se encontra, ao passo que "const" permite a declaração de variáveis cujos valores não deverão ser alterados;
  • "//" indica um comentário de uma só linha, enquanto que "/*" e "*/" indicam um comentário de múltiplas linhas.

JavaScript - Arrays

Arrays são essencialmente sequências de valores entre dois parêntesis retos, "[]", e separados por vírgulas. A contagem dos seus elementos começa no valor 0.

Exemplo: var carros = ['Renault', 'Audi', 'BMW']

Declaração associada ao array do exemplo O que retorna a declaração
carros[0] 'Renault'
carros[1] 'Audi'
carros[2] 'BMW'

Algumas propriedades/funções associadas aos arrays (utilizando o exemplo anterior):

Propriedade associada ao array O que sucede
carros.length Retorna o número que representa o último índice do array (ou seja, a sua extensão)
carros.push('Opel') Adiciona 'Opel' ao array
carros.pop() Remove o último elemento do array e retorna esse valor ('BMW' neste exemplo)
carros.reverse() Inverte a ordem do array

JavaScript - Objetos

Objetos são estruturas "chave-valor" que se encontram entre parêntesis curvos, "{}", e, tal como nos arrays, são separadas por vírgula

Exemplo: var computador = {marca: "ASUS", memória: "8gb", idade_em_anos="3"}

Declaração associada ao objeto do exemplo O que retorna a declaração
computador.marca 'ASUS'
computador.memória '8gb'
computador.idade_em_anos '3'

Estruturas de controlo da linguagem JavaScript


Estruturas de controlo são utilizadas para controlar o flow do código, sendo as primeiras a que farei referência o "if" e o "else".

  • Se a execução presente no "if" for verdadeira, esta será executada, caso contrário o "else" será executado, podendo entretanto existir outra condição no scope das duas referidas (precisamente pelo meio), sendo esta indicada pelas palavras "else if".


  • A estrutura de controlo "switch" é muito útil para testar uma condição contra várias hipóteses, quando já se sabe à partida quantas estas serão. Um "switch" escolhe um de vários blocos de código a ser executado e termina quando o "break" (palavra reservada muito importante e que impede que caso uma das condições seja verdadeira, as que estão abaixo da mesma sejam executadas em sequência). o "default" é outra palavra reservada que, sendo não obrigatória, indica o que acontecerá caso nenhuma das hipóteses presentes anteriormente for verdadeira.


  • A estrutura "while" é executada enquanto uma determinada condição for verdadeira, enquanto que uma estrutura "do... while" garante que o loop irá ser executado pelo menos por uma vez. As palavra reservada "break" aborta a execução do while, enquanto que a palavra "continue" avança para a iteração seguinte do "while".

JavaScript - Funções


Em cada linguagem de programação, as funções possuem como objetivo obedecer ao princípio denominado de "DRY" ("Don't Repeat Yourself"). Funções são utilizadas para reutilizar código sem ter que escrever o mesmo novamente e podem tornar o código muito mais legível e "limpo". Poderão também ser passados argumentos que serão utilizados no corpo de cada função e há que ter a noção de que uma função poderá ou não retornar um valor. Existe a possibilidade de passar um número de parâmetros arbitrário utilizando o operador "...rest" que se traduzirá numa lista de argumentos presente num array. Funções poderão ser chamadas de diversas formas, da forma comum, isto é, com o nome da função e parêntesis curvos: "soma()" ou poderão ser chamadas dentro de um Event Listener (mais sobre este na secção que falará de interação do JavaScript com o HTML DOM), sendo que neste caso a função poderá ser chamada de "callback" ou de "anónima".

De que se trata o formato JSON?


Essencialmente, o formato JSON significa JavaScript Object Notation. Acerca do formato JSON:

  • Trata-se de um formato leve de troca de informação;
  • Poderá ser utilizado por qualquer linguagem de programação;
  • É auto-descritivo e de fácil compreensão.

Nota importante:

  • O formato JSON é sintaticamente idêntico à notação de objetos em JavaScript. Devido a este motivo, é possível converter objetos JSON em objetos JavaScript e vice-versa.

Método utilizado Resultado do método

let jsonString = JSON.stringify(obj) console.log(jsonString) // Imprime "{"nmrDeAluno":20170017}"
